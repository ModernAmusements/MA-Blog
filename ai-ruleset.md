# AI Ruleset

## Workflow
1. **Write code** → 2. **Review code** → 3. **Reason code** → 4. **Fix code** → 5. **Repeat**

## Code Quality
- Always write production-ready code
- Never leave TODO comments
- Use proper TypeScript types
- Follow consistent naming conventions

## Communication
- Be concise, answer directly without preamble
- Use code references with file:line format
- Ask questions only when blocking

## Best Practices
- Use existing codebase conventions
- Write self-documenting code (no comments unless complex logic)
- Test critical functionality
- Verify build passes before considering done
